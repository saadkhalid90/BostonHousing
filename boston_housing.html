Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 always fails to predict the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is no better than one that naively predicts the mean of the target variable.

Question 3 - Training and Testing¶

What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?
Hint: What could go wrong with not having a way to test your model?

Answer:

While fitting models using machine learning algorithms, we always have to consider the bias-variance tradeoff. We not only want to explain the variance within our data, but also want to predict well on new data. Therefore, in order to generalize well to new/ unseen data, we want to avoid 'overfitting' or high variance/ low bias fits. Splitting the data into training and test datasets gives us a great way to evaluate the performance of our learning model as we only use a fraction of our data to fit the model and evalute the fit model by making predictions on unseen data.

Question 7 - Grid Search¶

What is the grid search technique and how it can be applied to optimize a learning algorithm?

Answer: Grid search is a method to evaluate the testing performance of a machine learning algorithm over a grid/ possible combinations of tunable parameter values. We evaluate a score through a specified scorer function for each possible combination of parameters and eventually chose the parameter combination with maxmimum testing score. The approach, however can only be implemented for a discrete parameter space (parameters taking on discrete values). Even for continuous tunable parameters, we can test only for discrete values through the grid search method.

Question 8 - Cross-Validation¶

What is the k-fold cross-validation training technique? What benefit does this technique provide for grid search when optimizing a model?
Hint: Much like the reasoning behind having a testing set, what could go wrong with using grid search without a cross-validated set?

Answer: We can use grid search using the validation set approach; by allocating a fraction of our data for training a model and the remaining data for testing performance. However, this approach has a disadvantage in that the split reduces data points for fitting a model which may cause high bias (as shown by the learning curves). To solve this issue, k fold CV can be used in which the data instead of two parts is equally divided into k - folds. A model is fit on data from k-1 folds, whereas a performance metric is computed using the remaining fold. This process is repeated k times to get k performance metrics which can finally be averged to summarize the cross validation performance. So instead of using the validation approach, we can use cross valiation while doing implementing the grid search for optimal parameters.

Moreover, validation approach (only a single split dividing the data into training and testing sets) scores/ errors are very sensitive to the nature of the split. Cross validation does not suffer from this disadvantage as the model is trained and evaluated k times with different overlapping training sets and non overlapping test sets respectively.

# TODO: Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor
from sklearn.grid_search import GridSearchCV
from sklearn.metrics import make_scorer

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth': range(1,11)}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)


    # TODO: Create the grid search object
    grid = GridSearchCV(regressor, params, scoring_fnc, cv = cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
